# Step 1: Feature Request Refinement

## Metadata
- **Status**: Completed
- **Original Word Count**: ~350
- **Refined Word Count**: ~450

## Refined Feature Request

Build the MVP Web UI for the Skills Marketplace and introduce a new bundle entity across the full stack. On the API side, add a `bundles` table to `packages/api/src/db/schema.ts` with columns for id, name, description, githubPath, downloadCount, and uploadedAt, along with three join tables (`bundle_skills`, `bundle_agents`, `bundle_rules`) that link bundles to their contained items via foreign keys. Create corresponding bundle query functions in `packages/api/src/queries/bundle.queries.ts`, a bundle service in `packages/api/src/services/bundle.service.ts`, and bundle routes in `packages/api/src/routes/bundles.ts` supporting GET list with search, GET by id with populated items, GET download, and POST create. Modify the existing `upload.service.ts` batch upload flow to automatically create a bundle record when multiple items are uploaded together, linking the resulting skills, agents, and rules to that bundle. Add bundle Zod schemas to `packages/shared/src/schemas.ts`, corresponding types to `types.ts`, and include `'bundle'` in the `ITEM_TYPES` constant in `constants.ts`. On the web frontend, extend the API client in `packages/web/src/lib/api.ts` with functions for agents, rules, and bundles covering list, detail, and download operations. Create React Query hooks in `packages/web/src/lib/query/` for each entity type (`use-agents.ts`, `use-rules.ts`, `use-bundles.ts`) plus mutation hooks for uploads (`use-create-skill.ts`, `use-batch-upload.ts`), and establish a query key factory in `keys.ts`. Replace the placeholder home page in `packages/web/src/app/page.tsx` with a unified marketplace view using the existing `data-table.tsx` component, displaying all item types together with columns for type (rendered as a badge), name, description, download count, uploaded date, and an actions column containing copy-CLI-command and download buttons, along with a type filter dropdown, a debounced search input using the existing `use-debounced-value.ts` hook, column sorting, and URL-driven filter and search state via the existing nuqs integration. Create dedicated detail pages at `packages/web/src/app/skills/[id]/page.tsx`, `agents/[id]/page.tsx`, `rules/[id]/page.tsx`, and `bundles/[id]/page.tsx`, each showing full metadata, a rendered markdown preview of the main `.md` file content, a file list, copy-CLI-command and download buttons, with bundle detail pages additionally listing contained items as links to their individual detail pages. Build an upload page at `packages/web/src/app/upload/page.tsx` with a form supporting both single-item and batch upload, including name and description fields, a drag-and-drop file zone accepting `.md` files and `.zip` archives that leverages the existing `zip.ts` and `folder-detection.ts` utilities for automatic structure detection, API validation feedback, and post-upload redirection to the created item's detail page or the bundle page for batch uploads. Use the existing UI component library (button, card, input, badge, select) throughout and follow the established Tailwind CSS styling patterns defined in `globals.css`.
